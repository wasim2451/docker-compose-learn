What is Docker?

Docker is an open platform for developing, shipping, and running applications. Docker enables you to separate your applications from your infrastructure so you can deliver software quickly.

With Docker, you can manage your infrastructure in the same ways you manage your applications. By taking advantage of Docker's methodologies for shipping, testing, and deploying code quickly, you can significantly reduce the delay between writing code and running it in production.

Core Concepts

Understanding the fundamental building blocks of Docker

üì¶

Containers

A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.

üñºÔ∏è

Images

A Docker image is a read-only template with instructions for creating a Docker container. Often, an image is based on another image, with some additional customization.

üìÑ

Dockerfile

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Using docker build, users can create an automated build.

üîó

Docker Compose

Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application's services.

üíæ

Volumes

Volumes are the preferred mechanism for persisting data generated by and used by Docker containers. They are completely managed by Docker.

üåê

Networks

Docker networking enables containers to communicate with each other and with the outside world. Docker creates isolated networks for secure container communication.

Essential Docker Commands

The most commonly used Docker commands to get you started

Pull an Image

docker pull nginx:latest

Download an image from Docker Hub or another registry.

Run a Container

docker run -d -p 80:80 nginx

Create and start a container from an image.

List Containers

docker ps -a

List all containers (running and stopped).

Build an Image

docker build -t myapp:1.0 .

Build an image from a Dockerfile in the current directory.

Stop Container

docker stop container_id

Stop a running container gracefully.

Remove Container

docker rm container_id

Remove a stopped container.

View Logs

docker logs -f container_id

Fetch and follow the logs of a container.

Execute Command

docker exec -it container_id bash

Run a command inside a running container.

Writing a Dockerfile

A Dockerfile is a text document containing instructions to build a Docker image. Each instruction creates a layer in the image.

Best practices for Dockerfiles include:

  • Use official base images when possible
  • Minimize the number of layers
  • Use multi-stage builds for smaller images
  • Don't install unnecessary packages
  • Use .dockerignore file
  • Sort multi-line arguments

Dockerfile Reference ‚Üí

Dockerfile 
# Use an official Node.js runtime as base
FROM node:18-alpine

# Set the working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production

# Copy application code
COPY . .

# Expose port 3000
EXPOSE 3000

# Define the command to run
CMD ["node", "server.js"]
